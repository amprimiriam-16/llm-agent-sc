# Contributing to QA Chatbot Platform

## Getting Started

Thank you for considering contributing to the QA Chatbot Platform! This document provides guidelines for contributing.

## Development Setup

1. Fork the repository
2. Clone your fork
3. Create a feature branch
4. Make your changes
5. Submit a pull request

## Code Standards

### Python Code Style
- Follow PEP 8
- Use Black for formatting
- Use Ruff for linting
- Type hints required

### Commit Messages
- Use conventional commits format
- Examples:
  - `feat: add new MCP tool for analytics`
  - `fix: resolve embedding dimension mismatch`
  - `docs: update API documentation`

## Testing

Run tests before submitting:
```bash
pytest
```

## Pull Request Process

1. Update documentation
2. Add tests for new features
3. Ensure all tests pass
4. Update CHANGELOG.md
5. Request review

## Questions?

Create an issue on GitHub for questions.
